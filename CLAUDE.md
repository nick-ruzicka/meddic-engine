# MEDDIC Engine — Rules for All Terminals

## What This Is
Outbound signal intelligence system for  targeting financial institutions.
Monitors Twitter/LinkedIn/financial press for AI transformation signals, scores
contacts using Claude API + skill router, surfaces review queue for outreach.

## Architecture (mirrors Chariot Signal Engine)
- `database.py` — SQLite schema (firms, contacts, signals, scores, outreach_queue)
- `main.py` — Pipeline orchestrator (build this like chariot-reference/main.py)
- `collectors/twitter_collector.py` — TwitterAPI.io live signals
- `collectors/linkedin_collector.py` — Apify LinkedIn scraping  
- `scoring/contact_scorer.py` — Claude API + skill router scoring
- `enrichment/hunter_enricher.py` — Hunter.io contact finding
- `api/server.py` — Flask API for approve/skip/pipeline (like chariot-reference/api/server.py)
- `export/index.html` — Static dashboard, dark  theme, Tailwind inline
- `export/contacts_data.json` — Generated by scripts/update_dashboard.py
- `scripts/update_dashboard.py` — Reads SQLite → writes contacts_data.json
- `data/seed/seed_accounts.py` — 20 real firms, run this first

## Scoring Dimensions (4, not 3 like Chariot)
confidence = (0.30 × ICP_Fit) + (0.25 × AI_Readiness) + (0.25 × Reachability) + (0.20 × Signal_Freshness)

Score 75+ = Strong Match (Approve)
Score 55-74 = Good Match (Review)  
Score 35-54 = Moderate (Enrich)
Under 35 = Weak (Flag)

## Skill Router
The scoring call uses config/skill_router.py to assemble targeted prompts.
See config/skills/ROUTER_SKILL.md for full architecture docs.
The scoring skill file is at config/skills/scoring/icp_scoring_.md — DO NOT OVERWRITE.
